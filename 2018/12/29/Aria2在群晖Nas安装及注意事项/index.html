Aria2在群晖Nas安装及注意事项
Nas 群晖 分享 下载工具
Nas应用 2018-12-29

首先Aria2并不是很快，安装前必须明白这一点
在群晖中使用Aria2是因为曾经它通过浏览器插件配合能够高速下载百度云盘
现在百度云盘已经对接口流量进行了限制，当达到一定的流量水管自动变小

网络上已经有很多这方面的教程，但是坑还是存在
通过查看日志我们能发现问题的所在和一些参数的设置

比如看到了ERROR的关键字就说明了程序出现错误并且已经中断执行
到了WARN的关键字就说明了程序出现了警告并没有中断执行

查找选择镜像


我这里选择了星号最多的一个
因为星号越多说明喜欢的人越多
那么镜像的质量相对来说就越好

在选择好镜像后就要对镜像的文档进行查看

进入文档后要看它相关的一些参数设置

Install
I. replace /DOWNLOAD_DIR and /CONFIG_DIR for save data, and YOUR_SECRET_CODE for security. Run command below
可以看到这里说需要将下载路径和设置路径转换为docker对外地路径/DOWNLOAD_DIR和/CONFIG_DIR就是关键字，代码下就出现了如上的关键字

下面的其实是命令行的各种参数，我们要将这些数据兑换成Nas的图形界面参数

sudo docker run -d \       这句的意思就是以高权限的身份运行docker并且后台执行 \反斜杠是命令行换行的意思
--name aria2-with-webui \ 自定义镜像名称
-p 6800:6800 \  docker内部端口6800对应外部端口6800
-p 6880:80 \    意思同上 左边为本机端口 右边为docker提供的端口
-p 6888:8080 \  意思同上 docker提供的端口是可以重复的 但是本机端口只有一个
-v /DOWNLOAD_DIR:/data \  docker的外置下载路径  /data 是内部的路径
-v /CONFIG_DIR:/conf \    docker的外置配置路径  /conf 是内部的路径
-e SECRET=YOUR_SECRET_CODE \  加密字符串
xujinkai/aria2-with-webui  原镜像的名称
II. Open http://serverip:6880/ for aria2-webui, open http://serverip:6888/ to browse data folder.
这里就是告诉你 webui 网页图形界面功能 是对应着这个6880端口 内部是80
下载资源界面 对应着6888端口 内部是8080

从上面可以看到一些参数的命令
-p 配置端口
-v 配置挂载目录
-e 传递环境变量

就算看文档这里边还是有坑的
坑从何来？
路径外置这个路径一般我们都放在docker目录下
所以权限方面没有问题能读能写
如果不是管理员账户，但是有开放的docker应用权限
就要考虑一下写入权限的问题了
顺便一提在Linux下大多数情况都要先考虑权限的问题
另外一个坑待会再说
先上图形界面设置的截图

绕坑之路

好一切都按网络上教程走完了，坑也随之而来

IPv6无法绑定端口6800
