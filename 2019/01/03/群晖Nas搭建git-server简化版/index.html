群晖Nas搭建git server简化版
Nas 群晖 git
Nas应用 2019-01-03

一直以来搭建git server都很简单
为什么到了群晖上看到的教程复杂了许多
其实我觉得应该是更加简单和直观才对
对于一个注重自己隐私的开发公司公司来说
有自己的git服务器是一个非常不错的选择
不管是做Linux还是在Windows服务器下构建git服务器都是非常简单的
但是在群晖上构建主要阻碍来源于非administrator权限不能登录ssh的原因比较麻烦
构建起公匙列表储存就相对麻烦一些

安装套件

套件中心安装对应套件

给权限

控制面板开ssh


用管理员组的账号登陆ssh
Mac系统直接终端
Win可以安装git使用bash工具就能使用ssh
又不是运维谁那么常用的ssh，进去建好仓库即可

ssh 管理组用户名@群晖NasIP地址
输入密码
用户名@群晖名称：~$

ssh admin@192.168.1.10
admin@192.168.1.10's password:
admin@YouNas:~$

进到其中一块硬盘 都是volume开头数值结尾如：volume2
一定要是二级目录 当然可以提前建立好的
一级目录是没有权限的 普通账户无法建立 当然不建议非要这样环境建立在一级目录
之所以在volume里建立二级目录是为了方便 今后在群晖的webui网页的界面能够方便找到和赋权
那么就不需要在命令行里给这给那的权限

gitc是我的目录 请自己对应自己的

cd volume2/gitc
git init --bare sample.git

拉取推送地址格式 任选其一 我比较喜欢第二种

git clone ssh://[Git 用户]@[您的 Synology 服务器 IP 地址或主机名]/[Git 存储库路径]
git clone [Git 用户]@[您的 Synology 服务器 IP 地址或主机名]:[Git 存储库路径]

git clone git@192.168.1.10:volume2/gitc/sample.git

拉取完后就可以随便搞了
当然拉取的时候是需要输入密码的

密码方法方式

git clone
git push
git pull

这种方式不管是拉取还推送都要输入一次密码
只要文件路径和git server都给了访问写入权限就可以正常拉取
非常方便，不想给用户权限了关闭即可，不用进命令行操作

公匙访问方式

使用免密登录的风险太高
这里解决不推荐了
sudo -i用root的权限给自己一个和root一样的权限
然后新增一个.ssh/authorized_keys文件
写入公匙就可以免密登录

注意

只要是使用ssh都会有安全风险，要记得随用随关就好
