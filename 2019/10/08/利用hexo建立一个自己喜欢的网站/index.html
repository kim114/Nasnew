利用hexo建立一个自己喜欢的网站
hexo 博客 blog 建站
技术 情感 2019-10-08

前言

其实这种简书写东西还是非常不错的
果然在简书这类的平台还是，最终这些文章还是难逃被锁定封锁掉了命运
但是有些时候还是显得不那么方便，比如有时候要写些梯子相关的东西
或者对一些事物的强烈看法，似乎就不那么好了
又或者希望有自己喜欢的页面和样式等等
我比较喜欢有阴影圆角的图片样式
在这里就比较难以呈现出来
言归正传下面就开始

环境选择

首先我选择了webIDE,来建设静态站点
但是搭建放置页面程序我选择github
为什么呢？国内局域网有些东西还是很敏感的，不能涉及的东西还是有的
怕万一不小心哪天打了点擦边球就不大好了
选择放置国外的服务器会比较稳妥一些，而且容量方面也比较好

webIDE之前是coding的产物
现在给腾讯了，据说维护还是coding但是似乎话语权已经不在coding这边了
如果哪天真的出现了问题，那么真相只有一个

当然我们还有Cloud9这些更好的选择
以后有机会再说别的在线带环境的编辑器

webIDE

现在叫Cloud Studio微信登录即可
注册登录好以后新建一个空间

看到Hexo的环境了没？
勾选后创建即可
这个环境的在操作系统上的配置以后再说，到时我会放置个链接在这里
之所以选择线上已经不输好的环境就是为了避免重复配置环境的麻烦

还有另外一种的情况
如果你没仓库是不能直接选择运行环境的
那么可以这样
起个名称选择第一项，其实其他项目也行，建议选择第一项

然后进入空间后在选择操作环境
总共三个步骤

安装hexo

hexo 就是一个依靠node.js渲染出html页面站点的程序，再通过插件部署(上传)到服务器上

进入操作界面后
使用命令行，创建一个目录再进入它，目录名称是随便起的

mkdir hexo && cd hexo && hexo init

我这里的意思创建一个hexo目录然后进入这个hexo的目录
最后再这个目录下创建hexo脚手架
这一句命令就完成了，如果不明白什么意思，不要紧其实就Linux的命令和hexo的命令而已
但是注意了不能做根目录直接就hexo init命令，没有目录是直接搭建不了hexo的，必须新建一个目录
等待安装完成，当命令行就不再跳动字符了，也就是安装完成了
可以看到有目录下有这些文件
当然不是完全一样也不要紧，因为以前的hexo版本有些文件是没有的更新以后新增了些新的文件
文章是我提前写的，并不知道你现在在安装的时候会是哪个版本，但是只要能运行其实也都是一样的

不放心就运行一下这个命令

hexo s

出现这个地址就说明OK了

你可以通过右边栏上的访问链接就能看看站点的原始样式效果了

修改hexo

为了适应这个webIDE我们做一下简单修改更好运行些
找到根目录的/node_mudules文件夹

进去再找到hexo-server目录下的index.js
修改里面的port的端口4000为8080
主要是因为每次打开访问链接都会默认8080
而每次运行hexo s测试服务器都是4000端口，需要来回修改很麻烦也很容易就忘记
干脆就一次性改好，以后只管运行就好了

新增hexo后台插件

这个插件主要是为了后面能够愉快的书写文章而不用
使用hexo n 'title'这种命令，也是为了方便粘贴图片
当我们随意的一个截图操作就能方便的粘贴上文章上面去，而不用先上传图片这编写路径这样就非常麻烦
如果搞了那么多事情，恐怕连书写的欲望都没有了
言归正传
安装的命令是再进入hexo目录里面才执行的，别在目录外边就执行了
如果当前的命令行还出处于刚刚登录时候的根目录
就执行命令cd hexo进入目录，如果已经进入目录就不需要执行这条命令了
下面执行安装插件的命令

npm install --save hexo-admin-ehc

至于这个插件密码的问题以后再说
因为就你自己使用，其实不用密码也是可以的
后台的一些操作说明
篇幅问题就不在一篇文章里面讲完

部署到github

如何注册github，其实互联网上已经有很多这方面的文章可以参考
因为很简单
所以也放置到以后再详说
这里假设你已经有了注册好了

好，现在回到编辑器的命令行
先配置好全局的git名称和邮箱，否则push推不上服务器
执行两条指令，中文部分替换成你自己的
不是用来找回的信息，不用特别去记忆这段东西，可以理解为使用当前这个仓库的人是谁的，仅此而已

git config --global user.name “用户名”
git config --global user.email “邮箱”

开始生成公匙
先在编辑器的命令行配置生成公匙

ssh-keygen -t rsa

点击Add SSH key新增公匙即可

好，开始新建个仓库
还是页面的右上角有个+符号

选择New rpository

新建完成后，进入这个仓库
选择SSH按钮，复制后面的这段git推拉（上传下载）地址，后面会在配置hexo文件中使用到

github的最后一步，还要回来设置GitHub Pages
这里先到webIDE编辑器中完成hexo的设置再推送一次代码，让其拥有一条分支再做GitHub Pages的设置
先切换到webIDE
命令行进入hexo目录
安装一下git推送插件,注意一定要安装否则你推送不了代码,有几次用来重复的代码让我以为不用安装这个插件导致推送的时候出现问题

npm install hexo-deployer-git --save

然后再找到你的hexo目录下的_config.ymlhexo配置文件
我的是/hexo/_config.yml 这个目录
双击打开它，并且滚动到最后的项目
deploy:这项编辑

deploy:
  type: git   
  repo: 刚刚复制到git的推拉（上传下载）地址
  branch: 分支名称（默认建议填上master,官网文档说github会自动检测不写也行，我没试过）

运行hexo指令完成一次推送

hexo g && hexo d && hexo clean

指令的意思是先渲染出文件
然后再推送渲染的文件到github
最后再清理渲染的文件，节省空间
配置完成后，webIDE的部分就没了

最后回到刚刚没有设置好的那部分，就是设置GitHub Pages
点选Settings设置仓库

往下滚动滚动条找到GitHubPages
按照如下设置即可
