centOS下配置hexo环境
centOS nas 群晖
docker 2019-02-01

hexo是一个非常好用的blog程序
本站就是基于hexo生成的静态页面
要知道hexo也是基于node.js来做出程序
那么node环境就一个必备的

在这里还是要使用到docker来构建node环境
这样就不会每次重装系统或者在外地想写东西的时候要搭建一次环境
之前我写过一篇使用webIDE来搭建hexo环境的文章
但是webIDE始终都是放置在别人的服务器上
难保有一天它也保不住，因为这webIDE我都已经看见换了两家老板
就算它保住了也避免不了各种广告要求
靠天吃饭虽然也要，但是自力更生才是王道

基于nas构建hexo环境
选择系统可以是centOS或者Ubuntu都可以，但是不推荐太精简版本的镜像
太精简的镜像，会导致你在扩展些其他的内容的时候出现各种莫名的问题

下载node

ubuntu更新列表后下可以直接安装

apt update
apt install nodejs
node --version

centOS就曲线救国吧

yum update
yum install epel-release
yum install nodejs
node --version

下载hexo

npm install hexo-cli -g

安装hexo

先找一个适合的目录安装并启动一下看看

hexo init blog
cd blog
npm install
hexo s

远程访问思路

首先远程访问你得有一台nas的服务器
不管是群晖的nas还是什么牌子的nas，只要支持docker就行
为什么说nas不说服务器呢？谁写个blog要往家里放服务器？
我当初往家里放服务器，不仅风扇吵得要命，电费也稀里哗啦

因为我要远程操作hexo所以要挂载一个文件夹路径给docker的hexo环境
主要是方便在外部修改hexo的主题文件和配置文件等等
如果你vim玩得溜也可不用挂载文件出来
但是一定一定要记得用远程仓库备份否则容器一删掉就没了
Linux下安装git都很简单，除非你用的是精简版本的Linux，这也开头就建议不要使用太精简版本的原因
因为你要去下载好安装包再想办法传进容器中去，如果命令行还好操作一个mv进去了，nas上面的可视化界面就不好操作了
直接wget就好还传什么传？😂有些精简版本恐怕连wget都得想办法想传进去才能用

apt install git
yum install git

hexo环境负责渲染静态页面和后台服务
群晖nas本身web station或者可以建立一个docker的web服务器对外发布
外网端口映射和web station的设置这里就不讲了我的其他文章里面有

这里还有一个先说的地方既然是远程写blog
当然就是想写的时候就能写
所以hexo中的hexo-admin后台插件就必不可少
能进得了nas自然就能进得了后台插件写blog了
启用了后台插件那么测试服务器的hexo s的默认端口是4000
就需要做一个外网转发和docker内部映射
另外这个端口是可以修改的在hexo-server里面的index.js里面修改
hexo-server在node依赖组件文件夹里面

  1. docker终端中启动hexo s
  2. 外网访问转发的hexo-admin后台写文章
  3. docker终端中渲染页面 hexo clean && hexo g
  4. 如果要外挂仓库的静态页面加速 hexo d 前提是配置好hexo文件

关于hexo的那些坑以后再说

差点忘记了 ，如果不传远端仓库做静态网页webpages的话
就把网页路径指向hexo下的public目录下
渲染前清除一次hexo clean之前的hexo版本
不清除渲染出来的还是之前的页面更新没有效果就是因为这样的坑
