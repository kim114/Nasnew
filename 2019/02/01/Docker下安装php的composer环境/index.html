Docker下安装php的composer环境
nas centOS
2019-02-01

关于composer安装主要问题会集中在root账户和php环境上
解决root账户下拉取代码的问题其实官方虽然有解答但是并不好用
php的问题主要集中在版本太高无法拉取的错误比如最新的7.3.1环境拉取代码的时候偶尔会出现抽风情况
这里就废话一下
为什么要搭建这些环境
因为如果经常换电脑、经常使用不同的操作系统、经常重装系统、经常换不同的语言编程
你会知道配置一个环境是多么繁琐的一件事情
另外你也会知道不是谁的电脑上都让你装各种环境工具
搭建一个可以长期使用环境服务器只要用ftp来修改代码要方便许多许多

Docker系统选择

首先如果是群晖nas不建议直接选择composer镜像或者是php镜像来用
因为有参数加入看composer镜像镜像的文档就知道要写composer.json文件来拉取代码
但是我们通常的编程过程中都是用到什么就拉取什么
很少为了拉取代码去编写composer.json文件然后还创建一个容器来拉取一小段代码完了再删除掉容器
免得太多容器自己都搞不清楚用那个容器了
直接用php镜像来构建composer环境你会发现root账户会是你的阻碍
php镜像是用了精简版本的Linux环境不管是新增用户还是给用户权限都是很麻烦的
这里推荐使用dockerhub上的centOS或者Ubuntu创建非root用户来搭建php环境

PHP版本选择

可以选择安装5.4版本以上到7.2版本以下
这样做起码稳定些不会在拉取数据的时候造成什么乱七八糟的问题

PHP版本安装

centOS使用默认版本即可目的主要是为了拉取代码而已
不用特地去搞各种没有必要的php组件
实在需要也可以去安装一下这些组件
起码比php镜像的那些Linux环境好办得多

yum update
yum install git -y
yum insatll php -y

Ubuntu也可以使用默认版本即可

apt update
yum install git -y
yum insatll php -y

真实的使用生产环境可定不是这些开发环境
生产环境的时候可以使用最新版本的php搭配集群nginx、集群数据库或者某些集成环境等等
docker的好处就在于你可以搭建很多的隔离环境来给自己的项目使用而不占用太多资源

安装 Composer

php -r "copy('https://install.phpcomposer.com/installer', 'composer-setup.php');"
php composer-setup.php
php -r "unlink('composer-setup.php');"
mv composer.phar /usr/local/bin/composer

新建用户

# 新建用户 可以不用建立密码
useradd -m username 
vi /etc/sudoers

sudoers文件增加的内容为： username ALL=(ALL:ALL) ALL
:wq保存退出

最后登录新建的账户
可以用版本命令查看了看一下是否报错

su username
composer --version

更新国内镜像

su username
composer config -g repo.packagist composer https://packagist.phpcomposer.com

之后都可以使用普通账户来拉取数据
不必没到一台机器上都要配置一次php在配置composer
远程终端操作就可以拉取到想要的代码
最后在绿色版本的ftp操作代码即可

当然也有例外比如有些强大的IDE工具自带环境光环就例外了
